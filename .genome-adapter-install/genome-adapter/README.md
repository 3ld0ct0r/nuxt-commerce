# Genome Storefront Adapter Drop

Framework: Nuxt / Nitro
Host target: Netlify
Project: prj_31c7daaa51c612f76d35fc41

This drop is generated by Genome as a server-side adapter seam. It contains no raw Genome tokens.
Install it into the storefront repository after onboarding, set the environment variables from env.example, and keep runtime and telemetry calls behind same-origin server routes.

Install placement:
- Copy src/genome-runtime.ts to server/utils/genome-runtime.ts.
- For Nuxt 4 app-directory projects, copy browser files under app/: app/utils/genome-render-model.ts, app/genome-target-map.json, app/components/GenomeRuntimeHome.vue, app/plugins/genome-runtime-slots.client.ts, app/utils/genome-telemetry.ts, and app/plugins/genome-telemetry.client.ts.
- For legacy Nuxt projects without app/, use the equivalent root utils/, components/, plugins/, and genome-target-map.json paths.
- Use framework/nuxt-nitro-server-routes.ts as the server-only Genome API handler under server/api/genome/runtime.ts. If you intentionally split manifest/config/telemetry Nitro routes, pass the split telemetry path into bindGenomeTelemetry instead of using its default endpoint.
- Use framework/nuxt-genome-runtime-home.vue, framework/nuxt-genome-runtime-slots.client.ts, framework/nuxt-genome-telemetry.client.ts, and framework/nuxt-genome-telemetry-plugin.client.ts as the runtime render and browser telemetry pattern.
- For Nuxt 4 app-directory projects, copy framework/nuxt-genome-runtime-page.vue to app/pages/genome-runtime.vue for add-only starter verification. For legacy Nuxt projects without app/, copy it to pages/genome-runtime.vue.
- Run framework/nuxt-install-target-slots.mjs after copying target-map.json into the chosen app root to install data-genome-slot attributes into confirmed Vue targets. Low-confidence target-map slots are skipped unless the operator confirmed them and a concrete file path exists. The Nitro runtime route pulling Genome is not enough unless the Nuxt page consumes /api/genome/runtime through a confirmed browser plugin and slot/component.
- Wire framework/nuxt-genome-heatmap-simulation-route.ts behind an operator-only or preview-only server/api/genome/heatmap-simulation.ts smoke route.
- Install framework/nuxt-nitro-self-test-route.ts at server/api/genome/self-test.ts.
- Install framework/nuxt-nitro-well-known-self-test-route.ts at server/routes/.well-known/genome/self-test.get.ts when the host exposes Nuxt server routes directly.
- On Netlify, append framework/nuxt-netlify.toml to the storefront root netlify.toml so /.well-known/genome/self-test rewrites to /api/genome/self-test.


Runtime pull:
- Fetch /api/v1/runtime-manifest with GENOME_RUNTIME_TOKEN on the server.
- Fetch /api/v1/runtime-config with GENOME_RUNTIME_CONFIG_TOKEN on the server for non-secret project capabilities.
- Resolve storage-backed manifest images through /api/v1/runtime-media with GENOME_RUNTIME_TOKEN on the server.
- Install the self-test endpoint at /.well-known/genome/self-test so Genome can verify the adapter after deployment.
- Build the render model with buildGenomeStorefrontRenderModel and render all media slots: hero, catalog_surface, product_featured, category_highlight, and mutation_candidate.
- For skeletal starter repositories, install the generated additive Genome runtime page instead of replacing starter homepage content.
- For Gatsby and Nuxt storefront pages, confirm target-map slots and add data-genome-slot attributes or wire the generated runtime component/hook into existing components; a serverless runtime proxy alone will pull Genome but will not mutate the visible page.
- Keep a last-known-good manifest fallback inside the storefront.

Telemetry:
- Forward browser events through the storefront server route.
- Bind data-genome-* surfaces with the generated telemetry client so coordinate heatmaps are available.
- Use the generated heatmap simulation runner for deterministic preview and checkout friction smoke tests. Simulation routes return dry-run events by default; they commit telemetry only when the server-side GENOME_HEATMAP_SIMULATION_TOKEN is configured and the request supplies the matching x-genome-heatmap-simulation-token header.
- Do not expose GENOME_RUNTIME_TOKEN to browser JavaScript.
